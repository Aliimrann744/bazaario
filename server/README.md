# Bazaario ‚Äî Server (Marketplace API)

Modular Node.js / Express backend for the Bazaario marketplace. Implements the Release‚Äë1
(Classifieds MVP) scope of `marketplace-implementation-blueprint.md` with a **config‚Äëdriven category
engine** powered by `category-schema-reference.json`.

## Stack
- Node 20 + Express
- **Prisma ORM + PostgreSQL** (jsonb attributes, BigInt money)
- JWT auth (access + refresh), bcrypt password hashing
- zod request validation

## 1. Create the database in pgAdmin 4

You need PostgreSQL installed and pgAdmin 4. Create the **role (user)**, **password**, and **database**.
Two ways ‚Äî pick one.

### Option A ‚Äî run the provided SQL script (fastest)
1. Open **pgAdmin 4** ‚Üí connect to your server as the **`postgres`** superuser.
2. In the left tree, click the **`postgres`** database ‚Üí top menu **Tools ‚Üí Query Tool**.
3. Open `server/prisma/init-db.sql`, paste it into the Query Tool, press **F5** to run.
   It creates role **`bazaario`** (password **`bazaario123`**) and database **`bazaario`**.

### Option B ‚Äî click through the pgAdmin UI
1. **Create the login role**
   - Left tree ‚Üí **Login/Group Roles** ‚Üí right‚Äëclick ‚Üí **Create ‚Üí Login/Group Role‚Ä¶**
   - **General** tab ‚Üí Name: `bazaario`
   - **Definition** tab ‚Üí Password: `bazaario123`
   - **Privileges** tab ‚Üí turn ON **Can login?** and **Can create databases?** ‚Üí **Save**
2. **Create the database**
   - Left tree ‚Üí **Databases** ‚Üí right‚Äëclick ‚Üí **Create ‚Üí Database‚Ä¶**
   - **General** tab ‚Üí Database: `bazaario`, Owner: `bazaario` ‚Üí **Save**

> Using different names? Change all three (user / password / database) consistently in
> `server/.env` ‚Üí `DATABASE_URL` (see below).

The connection string the app expects (already in `.env.example`):
```
DATABASE_URL=postgresql://bazaario:bazaario123@localhost:5432/bazaario?schema=public
```
Format: `postgresql://USER:PASSWORD@HOST:PORT/DATABASE?schema=public`
(default host `localhost`, default port `5432`).

## 2. Configure & run the API

```bash
cd server
cp .env.example .env          # then edit DATABASE_URL if you changed user/pass/db
npm install                   # installs deps + runs `prisma generate`

# Create all tables from the Prisma schema, then load demo data:
npm run prisma:migrate        # -> name it "init" when prompted   (creates prisma/migrations)
npm run seed                  # loads categories, locations, ref data, demo users + listings

npm run dev                   # http://localhost:4000   (or: npm start)
```

Or do migrate + seed in one shot:
```bash
npm run setup                 # = prisma generate + prisma migrate dev --name init + seed
```

Health check: `GET http://localhost:4000/health` ‚Üí `{ ok: true, db: "ok", ... }`

## 4. Architecture
```
prisma/
  schema.prisma      all models + relations (PostgreSQL, jsonb, BigInt money)
  init-db.sql        pgAdmin bootstrap (role + database)
  migrations/        generated by `prisma migrate`
src/
  config/            env config
  db/
    prisma.js        shared PrismaClient singleton
    seedData.js      taxonomy, locations, reference catalogs (static)
    seed.js          truncate + load demo data (npm run seed)
  schema/
    categorySchema.js  THE engine: loads category JSON, builds form schemas,
                       validates + normalizes attributes, derives facets
  middleware/        auth (JWT), error handler (maps Prisma P2002/P2025/P2003)
  modules/           one folder per domain (auth, listings, search, ...)
  utils/             ApiError, serializers, helpers, asyncHandler
  app.js / server.js wiring + boot
data/
  category-schema.json   copy of the provided schema reference
```

## 5. The category engine (why this matters)
Every category's form fields, validation, filters and sort options come from
`data/category-schema.json` ‚Äî **not** hard‚Äëcoded. Adding a category/option is a data change.
- `GET /v1/categories/:id/form-schema` ‚Üí the exact fields the client should render.
- On `POST /v1/listings`, attributes are validated + normalized against that schema
  (required, min/max, options, `current_year_plus_1`, salary `gteField`, computed
  `area_square_metres` from marla/kanal/sq‚Äëft, money in minor units).
- `GET /v1/search` derives **facets** from `facetable` fields automatically.

## 7. Notes & troubleshooting
- **`Environment variable not found: DATABASE_URL`** ‚Üí you didn't copy `.env.example` to `.env`.
- **`P1000 / authentication failed`** ‚Üí user/password in `DATABASE_URL` don't match the role you
  created in pgAdmin.
- **`P1001 / can't reach database server`** ‚Üí PostgreSQL isn't running, or wrong host/port.
- **`permission denied for schema public`** (PostgreSQL 15+) ‚Üí run the two GRANT lines at the bottom
  of `prisma/init-db.sql` while connected to the `bazaario` database.
- Money is stored as **BigInt minor units** (paisa); the API serializes it to a JS number in responses.
- Schema is managed by **Prisma Migrate** ‚Äî never hand‚Äëedit tables; change `schema.prisma` and run a
  migration. Production uses `prisma migrate deploy`.
